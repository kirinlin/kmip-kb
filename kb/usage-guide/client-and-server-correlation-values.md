# Client and Server Correlation Values

## Overview

KMIP provides two correlation value fields for request tracing. The Client Correlation Value is a client-supplied tag (non-unique, for informational purposes); the Server Correlation Value is a server-generated globally unique identifier for each request. Both appear in server logs, enabling end-to-end request tracing across client and server audit records.

## Guidance

The Client Correlation Value is used by the client to annotate requests with application-level context — such as a KMIP test case number, a transaction identifier, or a workflow ID. The server logs this value but does not act on it; it is purely informational.

The Server Correlation Value is generated by the server and returned in every response. Clients should store it in their own logs. Ideally, both sides can locate all log records containing a given Server Correlation Value to reconstruct the complete workflow associated with a request.

## Implementation Notes

The Server Correlation Value is the primary tool for correlating client-side and server-side logs in incident investigations. Deployments should implement structured logging with the Server Correlation Value as a searchable field. The Client Correlation Value is useful for correlating KMIP activity with application-level audit trails that predate the KMIP interaction.
